Home Study Practice Exam Labs

Objective crosswalk

Use these labs after the related course lessons: baselines and wireless under 4.1; vulnerability work under 4.3; monitoring and logs under 4.4/4.9; identity under 4.6; incident response under 4.8; backups under 3.4; and risk or policy exercises under Domain 5.

Safety Rules

Run labs only on systems you own or are explicitly authorized to test. Prefer isolated VMs, disposable cloud sandboxes, or local containers. The purpose is defensive learning: observe, configure, document, and explain.

Lab 1: Build a Secure Baseline

Objective: Create a hardened Windows or Linux VM baseline.

  1. Disable unused services and default accounts.
  2. Enable host firewall rules for only required services.
  3. Apply updates and document the patch level.
  4. Record which settings are preventive, detective, or corrective.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 2: Firewall Rule Design

Objective: Practice least-permissive inbound and outbound rules.

  1. Define a web server use case requiring HTTPS from users and SSH/RDP from an admin subnet only.
  2. Implement the rules with Windows Firewall, ufw, or a lab firewall VM.
  3. Test allowed and denied paths from a second VM.
  4. Write a short change record with rollback steps.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 3: Wireshark Traffic Analysis

Objective: Capture and interpret normal traffic.

  1. Capture DNS and HTTPS browsing traffic.
  2. Use filters such as dns, tcp.port == 443, and ip.addr == x.x.x.x.
  3. Identify handshake patterns and DNS queries.
  4. Explain what is visible and what TLS hides.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 4: Log Triage and Timeline

Objective: Build a mini incident timeline from logs.

  1. Collect Windows Event Viewer or Linux auth logs from a test login session.
  2. Mark successful logins, failed logins, privilege use, and service starts.
  3. Normalize timestamps and note time zone.
  4. Write a five-line incident timeline.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 5: Vulnerability Management Drill

Objective: Scan and prioritize findings in an authorized lab.

  1. Run a local scanner or review sample CVE findings.
  2. Rank findings by CVSS, exploitability, exposure, and asset criticality.
  3. Choose top three remediations.
  4. Create a patch/change plan with validation steps.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 6: Password and MFA Policy Review

Objective: Evaluate identity controls against common attacks.

  1. Draft a password/MFA standard for employees and administrators.
  2. Map controls to password spraying, credential stuffing, and phishing.
  3. Define lockout or risk-based controls that avoid easy denial-of-service.
  4. Document exception handling.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 7: Email Authentication Check

Objective: Understand SPF, DKIM, and DMARC defensively.

  1. Use nslookup or dig to inspect TXT records for a domain.
  2. Identify SPF and DMARC records.
  3. Explain what DKIM signs and what DMARC alignment means.
  4. Write recommended policy progression: none, quarantine, reject.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 8: Backup and Restore Test

Objective: Prove recoverability rather than assuming it.

  1. Create test files and a backup using your OS backup tool or simple archive.
  2. Delete or alter the files.
  3. Restore to a separate location.
  4. Measure restoration time and estimate RTO/RPO implications.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 9: Incident Response Tabletop

Objective: Practice decision-making during ransomware.

  1. Define roles: incident commander, technical lead, communications, legal, business owner.
  2. Walk through detection, containment, eradication, recovery, and lessons learned.
  3. Decide when to isolate hosts, notify stakeholders, and preserve evidence.
  4. Produce an after-action report.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 10: Data Classification Exercise

Objective: Create handling rules for sample data.

  1. Classify public marketing copy, internal procedures, employee records, and payment data.
  2. Assign storage, encryption, sharing, and retention rules.
  3. Identify data owners and custodians.
  4. Define disposal requirements.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 11: Cloud Storage Review

Objective: Check cloud-style storage exposure concepts.

  1. Create a mock inventory of buckets/shares and sensitivity levels.
  2. Define public/private access requirements.
  3. List detective controls: access logs, alerts, CSPM checks.
  4. Write a remediation plan for an accidentally public restricted bucket.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Lab 12: Forensic Evidence Handling

Objective: Practice evidence documentation without touching real incidents.

  1. Create a test file and calculate its hash.
  2. Copy it as evidence and hash the copy.
  3. Fill a chain-of-custody record with handler, time, purpose, and storage location.
  4. Explain why analysts avoid modifying originals.

Deliverable: one-page notes with screenshots or command output, risks found, and what control category each action supports.

Use only authorized lab environments.
