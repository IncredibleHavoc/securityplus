Objective 2.1 · 16 practice questions

Threat Actors

Threat-actor questions combine capability, access, motivation, and intent. Identify whether the actor is internal or external, skilled or unskilled, trusted or untrusted, and financially, politically, strategically, or personally motivated.

Core lesson

  • Nation-state, organized crime, hacktivist, insider, unskilled attacker, competitor, and shadow IT
  • Authorized versus unauthorized actors
  • Internal, external, and third-party access
  • Resources, sophistication, persistence, and institutional knowledge
  • Motivations: financial gain, espionage, ideology, disruption, war, revenge, and ethical research

Exam focus

  • A valid account does not make activity authorized.
  • Shadow IT is a governance and visibility risk even without malicious intent.
  • Nation-state attribution requires more than “advanced malware”; look for resources, persistence, and strategic objectives.

Threat Vectors and Attack Surfaces

A threat vector is the path used to reach a target; the attack surface is the collection of exposed opportunities. This section focuses on messages, files, voice, removable media, wireless, social engineering, impersonation, and compromised sites.

Core lesson

  • Message-, image-, file-, voice-, and removable-media vectors
  • Default credentials, exposed services, open ports, and vulnerable software
  • Phishing, spear phishing, whaling, smishing, vishing, and business email compromise
  • Impersonation, pretexting, invoice fraud, brand impersonation, and typosquatting
  • Watering-hole attacks and compromised trusted sites
  • Misinformation, social media, shoulder surfing, tailgating, dumpster diving, and elicitation

Exam focus

  • Spear phishing is targeted; whaling targets high-value executives or leaders.
  • Smishing uses text messages; vishing uses voice.
  • A watering-hole attack compromises a place the target already visits.
  • Choose the delivery path when asked for the vector, not the malware payload.

Objective 2.3 · 24 practice questions

Types of Vulnerabilities

Vulnerabilities are weaknesses that may be exploited. Focus on recognizing the flawed condition, not merely the attack result. The same compromise can originate from memory errors, insecure code, misconfiguration, obsolete hardware, cloud exposure, supply-chain compromise, or mobile-device changes.

Core lesson

  • Memory injection, DLL injection, buffer overflow, integer overflow, and race conditions
  • Malicious updates and compromised update channels
  • Operating-system, application, firmware, and hardware weaknesses
  • SQL injection and cross-site scripting
  • Virtual-machine escape, resource reuse, and hypervisor risk
  • Cloud misconfiguration, metadata exposure, directory traversal, and authentication bypass
  • Supply-chain dependencies, vulnerable libraries, and vendor compromise
  • Mobile rooting, jailbreaking, sideloading, and unsupported devices
  • Zero-day vulnerabilities and unavailable patches

Exam focus

  • SQL injection changes database queries; XSS causes script to execute in a user’s browser.
  • A race condition exploits timing or order of operations.
  • Directory traversal escapes the intended file path.
  • “Zero-day” describes patch/knowledge status, not severity.

Objective 2.4 · 34 practice questions

Indicators of Malicious Activity

The exam may provide symptoms rather than name the attack. Learn to connect artifacts, abnormal behavior, authentication events, network patterns, resource consumption, and user reports to the most likely malicious activity.

Core lesson

  • Ransomware, trojans, worms, viruses, fileless malware, spyware, keyloggers, rootkits, logic bombs, and bloatware
  • Physical attacks, RFID cloning, brute-force entry, and environmental attacks
  • DoS and DDoS resource exhaustion and amplification
  • DNS spoofing, poisoning, hijacking, and URL redirection
  • Wireless deauthentication, jamming, evil twins, and rogue access points
  • On-path interception, ARP poisoning, and traffic redirection
  • Replay, pass-the-hash, session hijacking, and token reuse
  • Privilege escalation, command injection, directory traversal, and malicious code
  • Cryptographic downgrade, SSL stripping, collision, and implementation attacks
  • Password spraying, credential stuffing, brute force, dictionary, and rainbow-table attacks
  • Indicators: lockouts, impossible travel, concurrent sessions, beaconing, unusual traffic, missing logs, and performance anomalies

Exam focus

  • Password spraying uses a small number of passwords across many accounts; credential stuffing uses known username/password pairs.
  • A worm self-propagates; a virus normally attaches to a host file or process.
  • On-path describes position; replay describes reuse of captured valid data.
  • An IOC is an observable artifact, not the full explanation of attacker intent.

Objective 2.5 · 18 practice questions

Mitigation Techniques

Mitigation questions ask for the control that most directly reduces likelihood or impact. Match the mitigation to the weakness: segment reachability, remove unnecessary exposure, enforce permissions, patch the vulnerable component, monitor for abuse, or apply a compensating control.

Core lesson

  • Network, application, user, and data segmentation
  • Access control lists and application allow lists
  • Least privilege and permission restrictions
  • Patching, firmware updates, and vulnerability remediation
  • Encryption, monitoring, EDR, and intrusion prevention
  • Disable unused ports and services; replace default credentials
  • Secure configuration baselines and hardening
  • Isolation, sandboxing, and compensating controls

Exam focus

  • Segmentation limits lateral movement but does not remove the original vulnerability.
  • A WAF can reduce web-attack exposure, but secure coding and patching address root causes.
  • Choose the narrowest effective control when the question asks for least privilege or least functionality.
